由 trigger 引发 Maximum call stack size exceeded

2016-02-21

为了完成dialog关闭后自动清除form表单数据，使用了bootstrap-dialog提供的事件

| Event | Description | Example |

|------------------|--------------------------------------------------|-------------------------------------------------------------------------|

| show.bs.modal | 在调用 show 方法后触发 | $('#modal').on('show.bs.modal', function () { /* do something */ }) |

| shown.bs.modal | 当模态框对用户可见时触发（等待 CSS 过渡完成） | $('#modal').on('shown.bs.modal', function () { /* do something */ }) |

| hide.bs.modal | 当调用 hide 实例方法时触发 | $('#modal').on('hide.bs.modal', function () { /* do something */ }) |

| hidden.bs.modal | 当模态框完全对用户隐藏时触发 | $('#modal').on('hidden.bs.modal', function () { /* do something */ }) |

这样当对话框关闭的时候能够及时的通知function清理，即使不去点击关闭按钮就能够知道dialog何时关闭。

现在正当我准备使用trigger完成提交后dialog的自动关闭事件hidden.bs.moda(我这里没有使用hide.bs.modal的原因是因为在表单数据足够多的情况下关闭的一瞬间有闪烁的现象)的时候，错误发生了：

堆栈空间已经超过已分配空间,这个错误在java能算是内存溢出了

解决

仔细的检查了我的代码，并没有使用循环/递归和定时器，然后发现我犯了一个非常低级的错误：

btn_sub.click(function(){              
    var ser = form.serialize();
     btn_sub.trigger("click");
});

这简直就是给自己挖坑啊！

Fix: dialog.trigger("click")

总结

后面在stackoverflow上面搜索了下，发现很多人都出现了这种错误，不过值得庆幸的是这种错误的发生基本都是犯了低级错误。

说下为什么后面trigger了click而没有使用hidden.bs.modal,因为我这里没有使用on进行事件监听,用了delegate：

$(document).detegate("div[role='dialog']", "hidden.bs.modal",
function() {

    var self = $(this);
    self.find("form").each(function() {
        $(this)[0].reset();
    });
});

补充 忘记了hide这个方法 换成了 dialog.modal('hide')

Ps：整站用了pjax 不用delegate不行啊。
